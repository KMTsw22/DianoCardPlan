# 테크트리 기획 — 개념 초안

> 메타 진행(Meta Progression) 축. 여러 Run을 거치며 **영구적으로 캐릭터의 기반 능력치·해금 풀**이 성장하는 시스템. Run 내 덱빌딩과는 별개의 장기 성장 축.
>
> **2026-04-28 개정**: 5브랜치 전용통화 모델 → 단일 포인트 + 4방위 + 랭크 노드(Last Epoch 톤)로 피벗. 본 문서는 새 모델 기준이며, 코드(`TechTreeCatalog.Nodes`)가 진실의 원본.

---

## 0. 한줄 요약

**조건 달성 → 단일 포인트 1 획득 → 중앙 루트에서 4방위로 펼쳐진 노드 중 하나를 클릭해 랭크업 → Run 영구 버프 적용.**
중앙 헥스 루트에서 **우=공격 / 좌=방어 / 상=경제 / 하=운영** 4방위로 가지가 펼쳐지고, 각 노드는 1~3회까지 누적해서 찍을 수 있음. 방위 끝에는 캡스톤 1개씩(브랜치 정수). 통화는 단일이라 자유롭게 분배 — **방향(=노드 위치)이 곧 정체성**.

레퍼런스: Diablo 4 Paragon Board / Last Epoch 어빌리티 트리 / Tiny Rogues 패시브.

---

## 1. 핵심 설계 원칙

1. **조건 = 실력 증명**. 랜덤 드랍 아님. "이걸 해냈으니 포인트 줌" 식의 명시적 보상.
2. **포인트는 단일 통화**. 어디든 자유롭게 투자 가능. ~~브랜치 전용 통화~~ 폐기 — MVP 단계에서 조건 시스템을 5종 만들 부담을 피하고 빌드 자유도를 우선.
3. **노드는 랭크형**. 한 번 찍는 게 아니라 **0/N 누적** — 같은 노드를 여러 번 찍어 효과를 쌓음 (Last Epoch 어빌리티 트리 모델).
4. **방향이 정체성**. 4방위(우/좌/상/하) 각각 테마가 명확. 같은 자유도를 줘도 *어디에 투자했는지*가 빌드 식별자.
5. **성장은 영구**. 찍은 랭크는 모든 후속 Run에 적용. 덱빌딩 성장(1 Run 한정)과 완전 분리.
6. **전제 사슬은 얕게**. N1(루트 인접)만 자유 해금, N2/N3는 N1 1랭크부터, 캡스톤은 N2 1랭크부터. 빠르게 캡스톤까지 도달 가능.
7. **스탯 버프는 온건**. 노드 1랭크당 +1~+5%. 캡스톤은 임팩트 있게 (예: 첫 턴 공격 ×2, 패배 시 1회 부활).

## 2. 4방위 구조

| 방위 | 테마 | 강화 축 | 톤 |
|------|------|---------|----|
| **→ 우 (공격)** | 정공법, 폭발 딜 | 공룡 ATK / 마법 피해 / 크리티컬 / 첫 턴 폭딜 | 적·주황 |
| **← 좌 (방어)** | 방벽, 생존 | 최대 HP / block / 도발 / 부활 | 청 |
| **↑ 상 (경제)** | 자원, 골드 | 시작 골드 / 상점 할인 / 보상 후보수 / 보스 골드 | 금 |
| **↓ 하 (운영)** | 핸드, 마나 | 시작 손패 / 시작 마나 / 정화 / 시작 유물 | 보라 |

> 라벨/연결선 색은 위 톤 기반. 캡스톤은 항상 글로우 펄스 + 옆에 큰 이름 라벨.

## 3. 포인트 획득

복잡한 조건 시스템 대신 **전투 클리어 = 즉시 지급**. 노드 등급별로 무게 차등.

| 트리거 | 보상 |
|--------|------|
| 일반 전투 노드 클리어 | **+1 pt** |
| 엘리트 노드 클리어 | **+2 pt** |
| 보스 처치 (챕터 14F) | **+3 pt** |
| 비전투 노드 (상점/휴식/이벤트/보물) | 0 pt |

> 패배(HP 0) 시 그 전투의 포인트는 미지급. 단, 그 Run 내 이미 획득한 포인트는 보존.
> 훈련장 모드에서는 포인트 미지급 (메타 진행 안 됨).

**한 Run 기대치**
- 14층, 평균 전투 노드 7~9 + 엘리트 1~2 + 보스 1
- 이론: 7×1 + 2×2 + 3 = **14 pt**
- 현실(끝까지 클리어): **10~14 pt**
- 보스 못 잡고 죽음: **3~7 pt** (간 만큼)

**페이싱**
- 한 방위 만점 13pt → **약 1 Run**
- 4방위 전부 만점 52pt → **4~5 Run**
- 풀 메타 ≈ Run 1회 30분 가정 시 **2~3시간**

> 너무 빠르면 조정: 일반 0/엘리트 +1/보스 +3 같은 변형. MVP는 풀 메타까지 빨리 가는 쪽이 첫 사용자에게 트리 시스템 매력 노출이 잘 됨.

## 4. 노드 구조

각 방위 = **N1 → N2 / N3 → N4(캡스톤)** 4노드 구성.

```
              [E1] - [E2] - [E4 캡]   (경제)
              [E1] - [E3] ─┘
                  │
[D4 캡] - [D2] - [D1] - [ROOT] - [A1] - [A2] - [A4 캡]   (방어/공격)
[D4 캡] - [D3] ─┘                [A1] - [A3] ─┘
                  │
              [U1] - [U2] - [U4 캡]   (운영)
              [U1] - [U3] ─┘
```

| 위치 | 코스트 | 랭크 | 설명 |
|------|--------|------|------|
| **N1** (루트 인접) | 1pt | 3랭크 | 작은 누적 버프. 해당 방위의 시작점. |
| **N2/N3** (분기) | 1~2pt | 1~3랭크 | 분기 — 한쪽으로 밀거나 둘 다 살짝씩 가도 됨. 캡스톤 prereq는 N2(메인 루트). |
| **N4 캡스톤** | 5pt | 1랭크 | 브랜치 정수. 강한 단발 효과. |

**전제(prereq)**:
- N1: 루트(가짜 노드, 항상 해금) → 자유 해금
- N2/N3: N1 1랭크 이상
- N4: N2 1랭크 이상 (N3로는 캡스톤 도달 불가 — 메인 라인 강제)

각 방위 만점: 1pt × (3+3+2) + 5pt = **13pt**
4방위 전부 만점: 약 52pt (장기 목표, 수십 Run).

## 5. Run과의 상호작용

**Run 시작 시** 적용:
- N1 ATK / HP / 골드 / 손패 등 영구 스탯 보정
- 시작 덱에 해금 카드 추가 (도발 / 정화 등)
- 캡스톤 시작 유물 (운영 캡스톤 U4)

**Run 중** 적용:
- 매 턴 block, 크리티컬 확률, 상점 할인 등 상시 패시브
- 첫 턴 공격 ×2 (공격 캡스톤 A4)

**Run 종료 시**:
- 조건 검사 → 포인트 지급 (Run 결과 화면에 누적 표시)
- 로비/탑바 어디서든 테크트리 UI로 진입해 클릭 — 즉시 저장

## 6. UI

- **진입**: 인게임 탑바 우측 (Deck 슬롯 왼쪽)에 절차적 트리 아이콘 슬롯. 보유 포인트가 1 이상이면 카운트 라벨 표시. 클릭 → 풀스크린 테크트리 화면 (현재 화면을 잠시 덮음, 닫으면 원래 화면으로 복귀).
- **레이아웃**: 1280×720 가상좌표, 중앙(640, 360) 글로우 헥스 루트.
- **노드 형태**: flat-top 헥사곤 + 안쪽 원형 글로우 + 방향별 절차 심볼(검/방패/동전/카드) + 노드 아래 `N/M` 랭크 카운터.
- **연결선**:
  - 잠김(전제 미해금): 회색 점선
  - 가능(전제 해금, 본 노드 미해금): 골드 점선
  - 해금(본 노드 1랭크 이상): 골드 실선
- **캡스톤**: 항상 펄스 글로우 + 옆에 큰 이름 라벨. 일반 노드는 툴팁으로만 이름·효과 노출.
- **상단**: "TECH TREE" 타이틀 + 우측 큰 글씨 "포인트: X".
- **하단**: ← 돌아가기 / [DEV] +1 / +5 / 전체 리셋 (조건 시스템 미구현 동안의 임시).

## 9. 데이터 / 영속화

- **저장 키**: PlayerPrefs `TechTreeState_v2` (JSON)
- **스키마**:
  ```json
  {
    "points": 7,
    "ranks": [
      { "id": "A1", "rank": 2 },
      { "id": "D1", "rank": 3 },
      { "id": "U1", "rank": 1 }
    ]
  }
  ```
- **마이그레이션**: 구 v1(브랜치 분리형) 데이터는 자동 마이그레이션 없음 — 빈 상태로 시작.
- **루트(ROOT)**: 카탈로그에만 존재하는 시각용 가짜 노드. 저장 데이터에는 들어가지 않고, `IsUnlocked` 호출 시 항상 true 반환.

## 10. 오픈 이슈

- [ ] 포인트 지급이 **자동 알림**(Run 끝 화면)인지 **로비 진입 시 일괄 표시**인지 — UX 결정

- [ ] 캡스톤 prereq를 **N2 only**로 강제할지 N3도 인정할지 — 현재는 N2만 (메인 라인 강제로 빌드 텐션 유지)
- [ ] 캐릭터별 트리 분리 여부 — MVP는 공용 1트리, 2챕터에서 캐릭별 시작 N1 가지(Arkane 전용 등) 추가 검토
- [ ] 조건 충족의 **Run 단위 vs 영구 누적** — 예: "도발 5회"가 한 Run 안인지 누적인지 (현재 안: Run 단위가 기본, "한 Run에서 X" 명시)

## 11. 다음 단계

1. **노드 효과 적용 로직** — `RunState` 초기화 시 `TechTreeCatalog.Nodes`를 순회하며 현재 랭크 × 효과를 반영. 시작 HP/마나/덱/유물/공룡 ATK 보정.
2. **Run 결과 화면 통합** — 이번 Run에서 획득한 포인트 합계를 보상/완주 화면에 표시 (현재는 누적 카운터로만 보임)
3. **밸런스 튜닝** — 한 Run당 포인트 양이 너무 빠른지 느린지 플레이 테스트 후 조정 (§3 변형안 참고)
4. **캐릭터별 분기** (2챕터 단계) — 시작 N1 옆에 캐릭터 전용 노드 추가 가능하도록 카탈로그 확장
