# 스타일 레퍼런스 (dianocard-web)

이 디렉토리의 이미지들은 Gemini 생성 시 **스타일 레퍼런스로 자동 첨부**됩니다.

## 두 가지 범위

### 1. 에이전트 공용 (`references/{agent}/*.png`)
해당 에이전트로 생성되는 **모든 엔티티**에 공통 적용. 화풍·톤·질감의 기본 베이스라인.

### 2. 엔티티 전용 (`references/{agent}/{id}/*.png`)  ← **신규**
**특정 카드·적·패시브 ID에만** 적용되는 전용 폴더. 있으면 **공용보다 우선**(공용은 스킵).
- 예: `references/card-art/C001/triceratops_ref.png` → C001(트리케라톱스) 카드 아트 생성 시에만 이 이미지 사용
- 예: `references/field-monster/E901/` → E901(이끼 수호석상) 전용

**의도**: 같은 에이전트라도 엔티티마다 원하는 레퍼런스가 다를 때 (T-Rex 카드에는 T-Rex 레퍼런스, 트리케라톱스에는 트리케라톱스 레퍼런스) 서로 섞이지 않게.

## 폴더 ↔ 에이전트

| 폴더 | 어디에 쓰임 | 엔티티 폴더 예 |
|------|-------------|----------------|
| `card-art/` | 카드 아트 | `card-art/C001/`, `card-art/C107/` |
| `field-summon/` | 맵 소환수 | `field-summon/C001/` (소환 카드 전용) |
| `field-monster/` | 맵 몬스터 | `field-monster/E004/`, `field-monster/E901/` |
| `card-effect/` | 카드 사용 이펙트 | `card-effect/C101/` (공격 마법) |
| `enemy-passive/` | 패시브 뱃지 | `enemy-passive/P_TWIN_RAGE/` |
| `icon/` | 프리폼 아이콘 | (엔티티 폴더 불필요 — 프리폼이라 slug로 구분) |
| `move-board/` | N프레임 모션 | `move-board/C001/` 등 |

## 포맷

- 확장자: `.png`, `.jpg`, `.jpeg`, `.webp`
- 권장 개수: 1–3장 (많이 넣어도 동작하지만 API 비용/토큰 증가)
- 파일명은 자유

## 추가/제거 방법

### UI에서 (권장)
각 에이전트 페이지(`/card-art`, `/field-monster`, `/enemy-passive` 등)에서:
1. 좌측 리스트에서 대상 엔티티 선택
2. 상단의 **엔티티 전용 레퍼런스** 박스에 이미지를 **드래그&드롭** 또는 **"파일 추가"** 버튼으로 업로드
3. 바로 썸네일이 뜨고, 이후 생성부터 이 레퍼런스가 적용됨
4. 썸네일 우상단 `×` 버튼으로 삭제

### 파일 탐색기로 (대량)
1. `references/{agent}/{id}/` 폴더 직접 생성 후 이미지 드롭
2. 웹 페이지 새로고침 → 자동 반영

## 우선순위 규칙

생성 시 레퍼런스 로딩 (위→아래, 첫 매치 사용):

1. **엔티티 폴더** `references/{agent}/{id}/` 에 이미지 존재 → 그것만 사용 (사용자 명시)
2. **자동 선택 (auto-refs)** → sub_type·enemy_type 기반으로 **기존 프로젝트 이웃 카드 아트**를 스타일 앵커로 자동 첨부 (최대 2장)
3. **agent 공용** `references/{agent}/` 폴더로 폴백

### 자동 선택 규칙 (lib/auto-refs.ts)

신규 카드·몹이 같은 결의 이웃 아트와 톤을 맞추도록 하드코딩된 매핑:

| sub_type | 자동 첨부 이웃 (최대 2장) |
|----------|-------------------------|
| ATTACK   | C101 공격 마법, C103 화염구 |
| DEFENSE  | C102 방어 마법, C104 수호 결계 |
| PURIFY   | C202 전체 힐, C104 수호 결계 (금빛/회복 톤) |
| HEAL     | C202, C205 재생 |
| TAUNT    | C203 광폭화, C206 분노의 함성 (붉은 오라) |
| ATTACK_BUFF | C201 공격 강화, C203 광폭화 |
| SPECIAL  | C206, C204 |
| STATUS   | C301 제물 의식 (음울한 톤) |
| DRAW     | C302 드로우, C303 발굴 |
| HERBIVORE | C001, C002, C003 |
| CARNIVORE | C004, C005, C010 |

적(field-monster)은 같은 챕터·같은 enemy_type의 이웃 → 같은 챕터 아무 이웃 순으로 자동 선택. 현재는 E001~E003만 아트가 있어서 챕터 1 몹들에만 실질적 효과.

### 바뀌는 체감

- **아무것도 안 해도** — 새 카드가 같은 서브타입 이웃 2장과 비슷한 톤으로 나옴
- **특정 카드만 원하는 레퍼런스 고정하고 싶으면** — 그 카드의 엔티티 폴더에 이미지 업로드 → 자동 선택이 꺼지고 사용자 이미지만 적용
- **agent 공용으로 전역 베이스라인 주고 싶으면** — `references/{agent}/` 에 팀 전체의 아트 스타일 샘플 1-2장 넣어두면 엔티티 폴더·자동 선택 둘 다 없을 때 폴백

## 주의

- 레퍼런스는 **스타일 힌트**로 Gemini에 전달됨. 프롬프트와 함께 결합되어 결과에 영향.
- 파이프라인에서 앞 단계 결과가 같이 첨부될 때는 **스타일 refs가 먼저, 체인 ref가 나중** 순서로 Gemini에 전달됨.
- 폴더 경로의 `{id}`는 CSV의 id와 정확히 일치해야 함 (대소문자 구분).
